Semantic and Non semantic Elements

Non semantic elements

Non-Semantic elements don’t have any meaning. They don’t tell anything about the content they contain. They can be used with different attributes to mark up semantics common to a group.

Following is the list of some non-semantic elements:

  • div
  • span

Semnatic Elements

These elements simply mean, elements with meaning. The reason being, there definition in the code tells the browser and the developer what they are supposed to do. Framing in simpler words, these elements describe the type of content they are supposed to contain.

Semantic Tag Description
article Defines an article
aside Defines content aside from the page content

aside Tag

The aside element represent the content which is indirectly giving information to the main content of the page. It is frequently represented as a sidebar.

Inside Aside tag

The aside tag is used to describe the main object of the web page in a shorter way like a highlighter. It basically identifies the content that is related to the primary content of the web page but does not constitute the main intent of the primary page. The aside tag contains mainly author information, links, related content, and so on.

Section tag

Section tag defines the section of documents such as chapters, headers, footers or any other sections. The section tag divides the content into section and subsections. The section tag is used when requirements of two headers or footers or any other section of documents needed. Section tag grouped the generic block of related contents. The main advantage of the section tag is, it is a semantic element, which describes its meaning to both browser and developer.

The section element is used to represent the standalone section within an HTML document. A page can have various sections and each section can contain any content, but headings for each section is not mandatory.
